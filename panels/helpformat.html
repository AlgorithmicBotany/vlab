Quick Help for VLAB Panel Manager

1 Panel Execution

1.1 Command Line
Execute Panel Manager using the following arguments:

>  panel [optional file name]
Opens a new panel, with optional file name.

>  panel [panel definition file]
Opens existing panel.

>  panel -m [colour palette file] [panel definition file]
Opens existing panel with specified colour palette.
(A colour map specification in the panel file overrides the command line argument. )

1.2 Activating Panel Items

Sliders

  • Select the outlined slider  between the title and value labels with the left mouse to grab focus.
  • Move the mouse left to decrease the slider value, and right to increase the slider value.
  • Release the mouse button to send the message.
        Or:
  • Click the left mouse on the small arrows on either side of the slider.  Click the left arrow to decrease the slider value, and the right arrow to increase the slider value.
Buttons
  • Select the button with the left mouse to grab focus.
  • Release the mouse button to send the message.
  • Monostable buttons will change colour while pressed, and return to the original colour after release.
  • Toggle buttons will switch between ON and OFF states (and colours) each time they are activated.
Button Groups
  • Button groups are sets of mutually exclusive toggle buttons.
  • If any button is activated, it becomes ON (even if it was ON previously) and the others are turned OFF.
  • It makes little sense to use monostable buttons in groups.
Menu Items
  • Press the right mouse anywhere on the panel to activate the popup menu.
  • Select the menu item from the popup menu to send the message.
Pages
  • Press the right mouse anywhere on the panel to activate the popup menu.
  • Select the page title from the popup menu to go to the page.
  • A message is sent to signal the page change.
1.3 Popup Menu
  • Raise always appears at the top of the popup menu. Its function is simply to bring the panel window to the front of the other windows on the screen.
  • Restore rereads the panel definition file to restore controls to their initial values, and sends all messages associated with the controls to stdout.
  • Reread rereads the panel definition file and redraws the controls in the panel window.
  • Broadcast sends all messages associated with the controls to stdout.
  • Edit opens the panel editor window, and switches to edit mode. A different set of menu options are then available.
  • Quit outputs the null character (end-of-file), which terminates the parameter editor , and closes the panel.


1.4 Interfacing with an Application

  • The panel manager can be interfaced to any application that has the ability to reread its data files.
  • Panel items print messages to standard output when activated with the click of the left mouse button.
  • A parameter editor is used to interpret the messages and modify the corresponding file. Two parameter editors are commonly used in the vlab system, ped and awkped.
  • The following example shows how the command line can be used to interface a panel with an application:

    •  

     

    > panel [panel definition file] | awkped [program file]

2 Panel Editor

In execute mode (default), activate the popup menu by pressing the right mouse button anywhere on the panel and select Edit to invoke the Panel Editor.
The panel editor is a separate window which interacts with the panel.  A 10x10 pixel grid is drawn over the panel, to aid with design and alignment.

2.1 Editing the Panel

The panel editor is divided into three main sections: the upper section contains buttons to add new items to the panel, the middle section contains horizontal and vertical alignment buttons for aligning items on the panel, and the lower section contains the editable fields of the panel.
The following panel fields may be edited:

  • Name: The panels name can be typed in the edit box.  The name is displayed in the panel window caption.  It is not the same as the panel file name.
  • Palette: A 256-entry RGB palette can be loaded by left clicking the button marked Load.  If no palette is loaded, panel colours are default.  If a palette is edited by left clicking the button marked Edit, the panel definition file must be reread by activating the popup menu and selecting Reread from File.
  • Font: A font for panel text can be specified by left clicking the button marked Set.
  • Width: / Height: The width and height for the panel can be entered into the edit boxes, or the panel ca be freely resized.  When resizing smaller, be cautious that all items on all pages remain within the panel bounds.


2.2 Panel Items

Labels, sliders, buttons and button groups appear on the panel.  Their contour is defined by a rectangular border around them and their origin is defined as the bottom left corner.  Pages and menu items appear in the panel's popup menu. The panel origin (0,0) is defined as the bottom left corner.

Adding New Items to Panel

  • Select an item from the group of buttons at the top of panel editor.
  • The new item automatically appears at origin (0,0) and is selected.
  • A popup dialog to edit  the new item is opened.
Selecting Items
  • Press the left mouse button over an item to select.
  • Double click the left mouse over an item to bring up the popup dialog.
  • The selected item is outlined in a colour contrasting the panel background.
  • The current page is selectable by pressing the left mouse on the panel background.  To go to another page, activate the popup menu by pressing the right mouse anywhere on the panel and select the desired page.
  • To select a menu item, activate the popup menu and select the desired menu item.  The popup dialog for the menu item is opened.
  • Once an item is selected, it may be moved, deleted, cloned or edited.
Moving Items
  • Once an item is selected, it may be dragged by holding the left mouse button while moving the mouse.
  • When the item's popup dialog is open, the origin may be entered into the edit boxes marked Origin:.
  • If a button group is moved, all buttons within it are moved.
Snap to Grid
  • To enable Snap to Grid option, activate the popup menu and select Snap to Grid from Edit Options.
  • When an item is selected or moved by with the left mouse, it will snap to grid upon release.  The bottom left corner will align with the nearest grid intersection.
  • Buttons in groups cannot be snapped to grid, however the group can be.
Deleting Items
  • Once an item is selected, it may be deleted by activating the popup menu by pressing the right mouse anywhere on the panel and selecting Delete Item from Edit Options.
  • A menu item may be deleted by clicking the button marked Delete on the menu item popup dialog.
  • A page may be deleted by activating the popup menu and selecting Delete Page from Edit Options.
Cloning Items
  • Once an item is selected, it may be cloned by activating the popup menu and selecting Clone Item from Edit Options.
  • A clone is exactly like the original item, and automatically appears at origin (0,0) and is selected.
Aligning Items on Panel
  • Once an item is selected, it may be aligned vertically or horizontally on the panel.  This is useful for panels designed as a single row or column.
  • Select an alignment from the groups of buttons in the middle section of the panel editor.
2.3 Editing Panel Items

An items popup dialog can be invoked by:

  • Double click the left mouse over the item.
  • Activate the popup menu by pressing the right mouse anywhere on the panel and select Edit Item from Edit Options.
  • Add a new item to the panel.
Each type of item has its own dialog.  The item type is specified  at the top of the dialog.  The close button is located at the bottom.
An items editable fields may include the following (not all items include all fields):
  • Name: The items name can be typed in the edit box.
  • Colours: If a colour palette has been loaded, the colour can be selected by pressing the right mouse button over the colour swatch and selecting a colour index on the popup palette.  If no palette is loaded, or no colour is selected,  the label beside the colour swatch displays a dash (-).  Otherwise, it displays the colour index between 0 and 255.  The first colour is the outline/text colour.  The second colour is the activation colour.  The third colour is the off colour.
  • Origin: The items (x,y) origin is displayed in spin boxes.  These values may be edited.
  • Min/Max: The range of sliders can be defined.  Min value MUST be less than Max value.
  • Value: The initial state can be defined for buttons and sliders.
  • Message: The message that is sent upon activation in execute mode must be specified.  As a reminder, the default message is: d <name> %d <scale>
Label Dialog
A labels editable fields include: name, colour, origin

Slider Dialog
A sliders editable fields include: name, colours (2), origin, min/max, value (between min and max), message

Button Dialog
A buttons editable fields include: name, colours (3), origin, value (on,off,monostable), message

Button Group Dialog
A button groups editable fields include:  colour, origin, buttons (list of names)
Buttons can be added to a button group by left clicking the button marked Add and then by selecting the button items on the panel.  When adding buttons to a group, only button items are selectable on the panel.
Left click the button marked Done when done adding buttons.
Buttons MUST have unique names, or they cannot be added to a group.

Menu Item Dialog
A menu items editable fields include: name, message

Page Dialog
A pages editable fields include: name, page label colour, page label origin, message
A pages name appears in its label.  All pages have labels, and page labels cannot be deleted, although they can be empty.

2.4 Executing the Panel

The panel can return to execute mode in the following two ways:

  • Activate the popup menu by pressing the right mouse anywhere on the panel and select Execute.
  • Close the panel editor by clicking the large button marked Close, located on the bottom of the panel editor.


2.4 File Options

  • Load opens a file dialog box to browse the users directories for the panel to load.
  • Reread  rereads the last saved panel definition file.
  • Save saves the panel.  If the panel file as not been specified, a file dialog will be opened to input the new panel file.
  • Save As opens a file dialog to input another file to save the panel as.

3 File Format

The Panel Manager requires a specific file format.  Panel definition files may be edited manually, or may be generated using the panel editor.  Each panel item requires certain specifications in sequence.  Each line in the file must begin with a heading, followed by a colon and a space separation.  The only optional specifications are the panel colour map: and the font:.  The following shows an example of a panel definition file, including all item types:
 
panel name: Model
colour map: map1.map
font: -urw-avantgarde-demibold-r-normal--17-120-100-100-p-0-iso8859-1
background: 36
size: 520 610

type: PAGE
name: Anabaena
color: 7
origin: 200 580
message: p 1

type: LABEL
name: Initial Cell
color: 7
origin: 133 555

type: SLIDER
name: Initial Conc (1/10)
colors: 7 1
origin: 20 340
min/max: 0 10000
value: 9225
message: n 4 1 -1 %d

type: BUTTON
name: Image
colors: 7 1
origin: 10 520
value: 1
message: s 1 image

type: BUTTON
name: 1st Split
colors: 7 1
origin: 10 460
value: 0
message: s 1 split

type: GROUP
color: 7
Image
L Justify
1st Split
ENDGROUP

type: MENU
name: Happy Message
message: happy
